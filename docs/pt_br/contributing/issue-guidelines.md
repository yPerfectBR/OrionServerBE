# Diretrizes de Issues

Abra issues separadas para problemas separados.

## Estrutura Recomendada

Em cada issue, descreva:

- o que aconteceu
- onde e quando aconteceu
- passo a passo para reproduzir
- comportamento esperado vs comportamento atual

## Extras Úteis

- Para bugs in-game: inclua versão do cliente e configuração do mundo/servidor.
- Vídeo curto (listado ou não listado) mostrando o erro acelera bastante o diagnóstico.

Quanto mais reproduzível e detalhada a issue, mais rápido tende a ser o processo de correção.
