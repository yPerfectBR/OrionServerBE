# Visão do Projeto

O OrionServer existe para oferecer uma base de servidor Bedrock custom com foco em controle total do que realmente roda em produção.

## Ideia Central

Um servidor não deve carregar recursos que não serão usados. O OrionServer foca no mínimo obrigatório para existir e deixa o restante como opt-in.

## Prioridades do OrionServer

- Runtime base leve
- Superfície de extensão para plugins e sistemas custom
- Desempenho e escala em cenários pesados no mesmo mundo
- Arquitetura que aproveite processadores modernos com múltiplas threads

## Casos de Uso Típicos

- Redes de minigames com lógica custom
- Projetos de mapa fixo (RPG, eventos, apocalypse)
- Servidores com demandas específicas onde paridade vanilla não é objetivo principal
